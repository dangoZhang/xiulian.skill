# 分享卡术语对照表

这份表专门服务默认分享卡文案。修仙写法只作为彩蛋备用。

原则只有三条：

- 先把话说清楚，再谈修仙味。
- AI 术语优先用常见中文写法，只有模型名、tokens、Agent 这类保留英文更自然时才保留英文。
- 用常见修仙小说语汇，不乱造体系。

## 卡片区块用语

| 卡片区块 | 建议标题 | 文案要求 |
| --- | --- | --- |
| 顶部标题 | vibecoding.skill | 固定写法 |
| 顶部标语 | 蒸馏你与 Code Agent 的协作记录 | 固定写法 |
| 主视觉 | 阶段 \| 等级 | 合成一行大字，只突出这两项 |
| 能力区 | vibecoding能力 | 要把能力讲全：这一层能做什么、这轮亮点、眼下短板；保持可截图传播 |
| 判词区 | 判词 | 只写“这一层的人，不一样在哪”；必须短而准 |
| 指引区 | 下一步 | 在下一级判词基础上继续细化，写成下一轮能照做的提醒 |
| 信息栏 | 模型 \| tokens | 只保留平台 / 模型和 token |
| 页脚 | 用户 \| 时间 | 信息要短 |

## 修仙术语 / AI 术语映射

| 修仙术语 | AI 术语 | 适用场景 |
| --- | --- | --- |
| 炼丹 | 提炼 / 蒸馏 | 提炼方法、总结套路、压缩经验 |
| 炼化 | 复用 / skill 化 | 把重复方法做成 skill / 模板 / 可复用法门 |
| 法门 | 工作流 | 一套稳定可重复的协作流程 |
| 章法 | 系统打法 | 更上层的组织方式与整体打法 |
| 分身 | Agent | 代为执行的一具或多具智能体 |
| 驭器 | 工具调用 | 调用工具、命令、搜索、读写文件 |
| 起手法诀 | prompt 调整 | 起手问法、引导方向 |
| 回炉 | 迭代修正 | 继续改 prompt、继续改要求 |
| 收功 | 结果验证 | 对结果做检查、测试、验收 |
| 回验 | 回看确认 | 回看改动、确认结果成立 |
| 观气 | 上下文阅读 | 看卷宗、看上下文、看环境来路 |
| 气脉 | 上下文承接 | 长回合续住主线 |
| 转圜 | 补救适配 | 遇阻后换打法、缩范围、补漏 |
| 行功 | 执行推进 | 真正开始做事、推进实现 |
| 入世 | 真实项目 | 进入真实场景、真实业务回路 |
| 回流 | 反馈回路 | 从真实结果再回炼到下一轮 |
| 传法 | 团队复制 | 把方法复制给团队或客户 |
| 法器 | 平台 / 模型 | 当前主用平台或模型 |
| 灵石 | tokens | 累计消耗的 token |

## 推荐文案骨架

| 区块 | 推荐写法 |
| --- | --- |
| vibecoding能力 | 现在已经能把熟悉任务沿着上下文稳稳推成多步协作，来回之间不容易掉主线。这轮最见功力的是协作节奏和补救适配。真正拦着往上走的，是结果验证和回看确认。 |
| vibecoding能力 | 现在开始把顺手打法沉成套路，同类任务会越做越快，越做越像自己的固定章法。亮点在起手清楚，短板在验证留痕还不够。 |
| 判词 | 这一层的人，常见任务能稳定推进到多步完成。 |
| 判词 | 这一层的人，开始把重复打法沉淀成可复用套路。 |
| 突破方向 | 下一层看的是能不能把顺手打法沉成可复用套路。 |
| 突破方向 | 下一轮先盯住结果验证和回看确认，把目标、执行、验证接成一条完整回路。 |

# vibecoding.skill 分享卡 Prompt

用于生成 `vibecoding.skill` 的默认分享卡。

默认风格是中性科技感，修仙风格只作为彩蛋：只有用户明确要求“修仙版 / 境界版 / xianxia style”时才启用。

## 目标

把你与 Code Agent 的真实交互记录，压成一张一眼能看懂、适合转发的能力卡。

这张图首先要像“分享卡”，其次才像“报告”。

## 默认标题

- title：`vibecoding.skill`
- slogan：`蒸馏你与 Code Agent 的协作记录`

## 默认版式

从上到下固定为：

1. title
2. slogan
3. `阶段 | 等级`
4. `vibecoding能力`
5. `判词`
6. `下一步`
7. `模型 | tokens`
8. `用户 | 时间`

版式上必须做到：

- 一眼先看到 `阶段 | 等级` 这一行主字
- 三个正文区块必须独立成块，不要连成一整页说明书
- 文字区要有明显留白，适合截图传播
- 最底部信息必须轻，不抢主信息

## 默认文案原则

- 先说人话，不要硬扯修仙
- 重点是“这一层通常能做到什么”
- 必须写出“这轮最亮眼的地方”和“最该补的地方”
- 只在用户明确要求时才把“阶段”翻译成“境界”
- 单个区块宁可更短更准，也不要写成长篇汇报
- 文案生成后要自查一遍：读起来必须顺口，像真人会发出去的总结

## 默认文案示例

- title：vibecoding.skill
- slogan：蒸馏你与 Code Agent 的协作记录
- 阶段 | 等级：稳定期 | L4
- vibecoding能力：现在已经能把熟悉任务沿着上下文稳稳推成多步协作，来回之间不容易掉主线。这轮最见功力的是协作节奏和补救适配：你一来一回接得顺，AI 遇阻也会换路继续推。真正拦着往上走的，是结果验证和回看确认：你还不够主动要证据和验收，AI 对收尾交代也还不够完整。
- 判词：这一层的人，常见任务能稳定推进到多步完成。
- 下一步：下一层看的是能不能把顺手打法沉成可复用套路。下一轮先盯住结果验证和回看确认，把目标、执行、验证接成一条完整回路。

## 视觉要求

- 整体要更像“等级卡”或“能力卡”，不要像浅色文档截图
- 外层背景使用等级色对应的深色渐变
- 内层主卡使用统一深色面板
- `阶段 | 等级` 是唯一超大字，其余文字保持统一字号
- 正文区块各自拥有独立面板、圆角和安全边距
- 不要添加意义不明的图案，不要堆视觉噪音
- 小标签、边框和发光只做辅助，不能喧宾夺主

## 彩蛋模式

当用户明确要求修仙版时，再切成：

- title：`vibecoding.skill`
- 左侧主字：境界
- 文案允许使用：闭关、破境、法器、分身、识海、传法
- 但模型名、tokens、Agent、workflow、prompt 这些词仍优先保留现代写法

# vibecoding.skill 分享卡 Prompt

用于生成 `vibecoding.skill` 的默认分享卡。

默认风格是中性科技感，修仙风格只作为彩蛋：只有用户明确要求“修仙版 / 境界版 / xianxia style”时才启用。

## 目标

把你与 Code Agent 的真实交互记录，压成一张一眼能看懂、适合转发的能力卡。

## 默认标题

- title：`vibecoding.skill`
- slogan：`蒸馏你与 Code Agent 的协作记录`

## 默认版式

从上到下固定为：

1. title
2. slogan
3. `阶段 | 等级`
4. `vibecoding能力`
5. `关键判断`
6. `下一步`
7. `模型 | tokens`
8. `用户 | 时间`

## 默认文案原则

- 先说人话，不要硬扯修仙
- 重点是“这一层通常能做到什么”
- 必须写出“这轮最亮眼的地方”和“最该补的地方”
- 只在用户明确要求时才把“阶段”翻译成“境界”

## 默认文案示例

- title：vibecoding.skill
- slogan：蒸馏你与 Code Agent 的协作记录
- 阶段：稳定期
- 等级：L4
- vibecoding能力：已经会把常见来回沉淀成工作流，熟悉任务通常能稳定推进到多步完成。这轮亮点在协作节奏和补救适配：你这边一来一回接得顺，AI 遇阻也会换路继续推。最该补的是结果验证和回看确认：还不够主动要证据和验收。
- 关键判断：当前处在稳定期 / L4。这一层已经能把常见任务稳定推进到多步完成。
- 下一步：下一轮先把目标、边界和验收写在前面，再让 AI 动手。每轮结束都补一句：改了什么、怎么验证、还有什么没验。

## 彩蛋模式

当用户明确要求修仙版时，再切成：

- title：`vibecoding.skill`
- 左侧主字：境界
- 文案允许使用：闭关、破境、法器、分身、识海、传法
- 但模型名、tokens、Agent、workflow、prompt 这些词仍优先保留现代写法
